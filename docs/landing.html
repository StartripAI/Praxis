The Problem

Models are the easy part.

Prediction libraries give you a model. Nobody gives you what your CFO, COO, and ops team actually need — the infrastructure around it.

01

Data ingestion hell

Data scattered across Excel, Feishu, and 5 different systems. No single truth store.

02

Inconsistent calibers

Every report calculates metrics differently. No versioned definitions, no governance.

03

Calendar misalignment

March 1 was Saturday last year, Sunday this year. Raw YoY comparison shows ±30% bias from calendar effects alone.

04

Hardcoded event effects

HOLIDAY_COEFFICIENT = 0.85 — somebody guessed this once. Nobody knows if it's still right.

05

Single-number forecasts

Management treats predictions as certainty. No range, no risk communication, no confidence intervals.

06

No verification culture

"Looks about right" is not validation. No backtest, no gate, no pass/fail threshold.

07

Growth is a black box

"Revenue went up" — but is it YoY? MoM? New stores inflating? Nobody decomposes it.

Architecture

Thin wrapper. Heavy leverage.

Praxis only builds the top two layers — business abstraction and adapters. Core algorithms are delegated to proven open-source backends, reducing effort by 60%+.

Praxis — Business Abstraction DM Mapping · Growth Decomp · Backtest Gate Event Learner · DOW Learner · Report Output ────────────────────────────────────── Praxis — Thin Adapters forecast/ · conformal/ · features/ ────────────────────────────────────── Open-source Backends (pip install) StatsForecast · Darts · MAPIE · tsfresh chinese-calendar · holidays · AutoTS ────────────────────────────────────── Data Layer DuckDB (star schema) · BigQuery/Snowflake MCP CSV · Excel · httpfs

Quick Start

One minute to learn.

# Install
pip install praxis-engine

# 1. Comparable trading days
from praxis.calendar import CalendarEngine
engine = CalendarEngine(country="CN", year=2026)
dm = engine.build_dm_mapping(target_year=2026, target_month=3)

# 2. Growth decomposition
from praxis.analysis.growth import GrowthDecomposer
g = GrowthDecomposer().decompose(115000, 100000, 110000)

# 3. Calibrated uncertainty (not a single number)
from praxis.forecast.conformal import ConformalWrapper
cw = ConformalWrapper(quantiles=[0.10, 0.50, 0.90])

# 4. Pass or fail — no ambiguity
from praxis.backtest.scoring import score
sc = score(actuals, preds, gate={"max_bias": 0.15})
